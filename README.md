# Miner Report

**Цель проекта** — получать отчёты о работе майнинговой фермы: хешрейт, температура чипов, потребление, аптайм и другие показатели по каждому устройству, сводка в Telegram.

Поддерживаются майнеры **Whatsminer** (API по TCP, порт 4028). Скрипт подключается к каждому VPN-профилю, опрашивает устройства, формирует Excel-отчёт и отправляет его в Telegram.

## Требования

- **Node.js** 16+
- **Windows** (используется `rasdial` для VPN)
- Майнеры **Whatsminer** в сети (доступ по API, порт 4028)
- Настроенные VPN-профили в системе (SSTP и т.п.)
- Telegram-бот и его токен

## Установка

```bash
npm install
```

## Настройка

1. Скопируйте шаблон окружения:
   ```bash
   copy .env.example .env
   ```

2. Отредактируйте `.env`:
   - `TELEGRAM_BOT_TOKEN` — токен бота от [@BotFather](https://t.me/BotFather)
   - `TELEGRAM_CHAT_ID` — ID чата или пользователя для отправки отчёта
   - `VPN_PASSWORD` — пароль по умолчанию (если у профиля не указан свой `pass`)
   - `VPN_PROFILES` — JSON-массив профилей (в одну строку), см. ниже
   - `MINER_PORT` — порт API майнера (по умолчанию 4028)

## Запуск

```bash
npm start
```

или:

```bash
node index.js
```

Скрипт по очереди подключается к каждому VPN, сканирует указанные диапазоны IP, собирает данные майнеров, создаёт `miner_report.xlsx` и отправляет файл с краткой сводкой в Telegram.

## Формат VPN_PROFILES

В `.env` переменная `VPN_PROFILES` — это JSON-массив объектов (в одну строку). Поля:

| Поле     | Описание              |
|----------|------------------------|
| `profile`| Имя VPN-профиля в ОС   |
| `user`   | Логин (по умолчанию `sstp`) |
| `pass`   | Пароль (необязательно; если нет — берётся из `VPN_PASSWORD`) |
| `ipStart`| Начальный IP диапазона |
| `ipEnd`  | Конечный IP диапазона  |

Пример одного профиля (пароль свой у профиля):

```env
VPN_PROFILES=[{"profile":"MyVPN","user":"sstp","pass":"mySecret","ipStart":"192.168.1.1","ipEnd":"192.168.1.255"}]
```

Без `pass` у профиля будет использоваться `VPN_PASSWORD` из `.env`. Несколько профилей — через запятую в одном массиве; у каждого может быть свой `pass`.
